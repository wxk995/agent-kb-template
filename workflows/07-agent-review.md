# Workflow: Agent 自检

## When to use

Agent 完成文档更新、规则调整、项目记忆维护或收口报告前使用。

## Read first

1. `AGENTS.md`
2. 本次涉及的 workflow
3. 本次修改过的文件
4. `standards/governance.md`

## Steps

1. 检查边界：
   - 是否改了未被用户确认的文件？
   - 是否把外部工具纳入默认流程？
   - 是否擅自提交、推送或创建 PR？
2. 检查事实：
   - 是否把未验证内容写成已完成？
   - 是否遗漏验证结果或剩余风险？
   - 是否引用了不存在的路径？
3. 检查结构：
   - 热层是否过长？
   - 新内容是否能被索引找到？
   - 同一规则是否重复写在多个文件？
4. 检查表达：
   - 是否有空洞总结？
   - 是否伪装成用户本人判断？
   - 是否把建议写成强制结论？
5. 运行可用的最小验证命令，例如 `git diff --check`。

## Done when

- 修改范围和用户确认一致。
- 路径、索引和 Markdown 格式通过检查。
- 结论有证据，未完成事项有说明。
- 最终回复能清楚说明改了什么、验证了什么、还剩什么。
