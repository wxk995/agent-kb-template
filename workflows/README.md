# Workflows

这里放的是可执行的操作流程，不是知识正文。

当你不知道应该读哪个文件、更新哪个文档、什么时候收口或怎么瘦身时，先从这里选一个 workflow。每个 workflow 都按同一结构写：什么时候用、先读什么、怎么做、做到什么算完成。

## 入口选择

| 场景 | 读取 |
|------|------|
| 第一次 fork 或阅读本模板 | `01-new-reader.md` |
| 给一个新项目建立记忆 | `02-new-project.md` |
| 新开 Agent 对话 | `03-start-agent-session.md` |
| 更新、补充、纠正文档 | `04-update-knowledge.md` |
| 工作结束后沉淀结论 | `05-closeout.md` |
| 文件太多、重复、过期 | `06-prune-knowledge.md` |
| Agent 写完后复查 | `07-agent-review.md` |

## 使用原则

- 先选 workflow，再读正文文档。
- workflow 只放步骤，不堆背景解释。
- 如果 workflow 和具体项目规则冲突，以项目规则为准。
- 修改 workflow 本身前，先说明要解决的问题和影响范围。
