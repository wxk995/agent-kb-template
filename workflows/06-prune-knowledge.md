# Workflow: 知识库瘦身

## When to use

用户说知识库太乱、太臃肿、找不到内容，或热层文件超过 50 行时使用。

## Read first

1. `standards/governance.md`
2. `AGENTS.md`
3. `workspace-index.md`
4. `global/experience-index.md`
5. 需要瘦身的项目热入口和相关温层文件

## Steps

1. 先盘点，不直接删除：
   - 热层文件行数
   - 重复规则
   - 过期状态
   - 没有索引入口的孤立文档
   - 临时输出或一次性材料
2. 按层级处理：
   - 热层只保留当前状态、下一步和硬规则。
   - 温层保留长期规范、验证、决策、错误、交接。
   - 冷层保存历史追溯内容。
3. 对重复内容，只保留一个权威版本，其余改为引用或列入归档候选。
4. 对过期内容，先列出归档清单和理由，等待用户确认。
5. 用户确认后再移动到 `archive/` 或删除临时内容。
6. 更新索引，确保入口能找到保留下来的权威文件。

## Done when

- 热入口重新变短。
- 重复规则有唯一权威来源。
- 旧内容不会继续误导新对话。
- 删除或归档动作都有用户确认记录。
