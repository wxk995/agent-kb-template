# 参与贡献

## 项目定位

这是从真实私人 Agent 工作知识库中抽离的轻量公开模板。它面向个人开发者，保持克隆即可使用，不是团队治理平台，也不是自动化 CLI 工具。

## 欢迎贡献的内容

- 规则纠错和表述澄清
- 模板结构和栏目改善
- 跨平台命令补充（PowerShell / Bash）
- 文档可读性和翻译改进

## 不接受的内容

- 私人业务资料、项目数据或客户信息
- 密钥、密码、Token 或任何认证凭据
- 聊天记录原文
- 不接受把特定 Agent 的专属能力写入公共核心规则，或复制多份平台规则；不重复规则正文的轻量入口适配器（如 `CLAUDE.md`）可以接受。
- 未经验证的大型重构

## 修改要求

- 先说明要解决的问题和影响范围。
- 默认启动链路保持 5 个热入口；单个热层文件超过 50 行时应先评估瘦身。
- 同一规则只有一个权威来源，其他文件只引用、不重复。
- 示例必须明确标注为虚构教学示例。
- 新增路径必须真实存在，或明确标注为占位路径（如 `<PROJECT_NAME>`、`<项目名>`）。

## 提交前验证

```bash
git diff --check          # 无空白错误
```

- 检查内部路径引用是否指向实际存在的文件。
- 扫描全文确认无私人路径、密钥、Token 或业务数据。
- 在 PR 或提交说明中写明验证结果。

## PR 说明

提交 PR 时请包含：

- **改了什么**：具体变更内容。
- **为什么**：要解决的问题。
- **如何验证**：跑过什么检查命令、结果如何。
- **剩余风险**：已知的未覆盖情况或潜在影响。
